音声認識サービスプロバイダの起動

SIGVerseシステムはjuliusを用いた音声認識サービスプロバイダを用意しています．エージェントのコントローラは，音声認識サービスにより，音声データから認識されたテキストを得ることができます．

SIGVerseシステムの用意している音声認識サービスプロバイダはsigjspです．sigjspは独立した実行形式のアプリケーションです．音声認識サービスを使用する場合は、シミュレーションサーバを起動した後でsigjspを起動させてください．

sigjspは，自分の存在をシミュレーションサーバに知らせるために，起動時にシミュレーションサーバへアクセスします．シミュレーションサーバのポート番号を以下のように指定して起動します．

	% sigjsp.sh -p 6500

ポート番号を指定しないと6000番ポートを使用します。


音声認識サービスプロバイダで使用可能はオプションは以下のとおりです。

	% sigjsp.sh [-s <serverName>] [-p <number>] [-sn <serviceProviderName>] [-sp <number>] [-mp <number>] [-ap <number>] [-e <path>] [-C <config.jconf>]

 OPTION :
	-s <serverName>  : server host name
	-p <number>  : server port number
	-sn <serviceProviderName> : sigjsp service provider name
	-sp <number> : sigjsp service provider port number
	-mp <number> : julius module mode port number
	-ap <number> : julius adinnet port number
	-e <path>	   : julius location (default: /usr/local/bin/julius)
	-C <config.jconf;> : config file for julius

sigjspはjuliusのモジュールモードを使って通信を行います．また、音声データはadinnetというjuliusの提供するネットワーク経由での音声入力ポートを使用します．モジュールモードでの通信ポート番号は-mpオプションで、adinnetでの通信ポート番号は-apオプションで指定します．

-eオプションはjuliusの実行形式のある位置を絶対パスで指定するために使います．またjulius自身に渡す設定ファイルは-Cオプションで指定します．例えば、

 % sigjsp.sh -e /usr/local/bin/julius-4.1.2 -C myconf.jconf

と書いた場合、sigjspは、
/usr/local/bin/julius-4.1.2 -C myconf.jconf
を実行してjuliusを起動します.

コマンドラインオプションを指定せず，sigjsp.confという名前の設定ファイルを使うこともできます.

デフォルトでは、sigjspはカレントディレクトリのsigjsp.confを探します．もし、環境変数SIGVERSE_JSP_CONF_DIRが定義されている場合、カレントディレクトリではなく、SIGVERSE_JSP_CONF_DIRディレクトリを探します．

sigjsp.confとコマンドラインオプションで同一の指定がされた場合は，コマンドラインオプションが優先されます．例えば、シミュレーションサーバのポート番号を、sigjsp.confと-pオプションの両方で指定した場合、-pオプションの値が使われます．

juliusに関する設定はjuliusのマニュアル等を参照してください
