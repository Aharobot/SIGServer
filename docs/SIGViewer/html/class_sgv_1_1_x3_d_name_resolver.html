説明

(SimObjのクラス名，状態値)と，(X3Dファイル名)の対応マップ

マップ構築のための入力は，以下のような文字列フォーマット． SimServerからのViewerAttach承認(ResultAttachView)と 共に送られてくる． （SimServerEventListenerrecvResultAttachView()を参照）

またはloadFromFile()により，csv形式のファイルから読み込むことも可能

[文字列フォーマットの意味]

"className:partsName:stateAttrName:stateAttrValue"

className ......... クラス名("Entity", "Agent", "Robot"等） partsName ......... パーツ名("Head", "Hand", "Foot"等) stateAttrName ..... 状態属性名 ("switch", "light"等） stateAttrValue .... 状態属性値 ("on", "off" 等)

なお、状態属性名と状態属性値を合わせた組を「状態値(stateValue)」と呼ぶ．

状態値の例： "light:on" "bottle:empty" ":" （＝「何もない」という状態）

（注1）大文字・小文字を区別する （注2）現時点ではパーツ名は使っていない

std::map< std::string, std::string > Sgv::X3DNameResolver::getX3DShapeNames ( const std::string &  className  ) 

クラス名から、X3Dファイル名を得る（複数の場合もあり）

返り値は「状態値」-->「X3Dファイル名」のマップ

引数:
className [i] クラス名
戻り値:
状態値とX3Dファイル名のマップ
 (例)
    "TV::switch:on" ---> "TV_on.wrl"
    "TV::switch:off" --> "TV_off.wrl"
    "apple:::" -----------> "apple.wrl"
    が登録されていた場合を考える．

    クラス名"TV"は2つの状態値"switch:on", "switch:off"を持ち
    クラス名"apple"は状態値を持たない

    このとき、

    std::map<std::string, std::string> ret1 = getX3DShapeNames("TV");
    ret1["switch:on"] ---> "TV_on.wrl"
    ret1["switch:off"] --> "TV_off.wrl"

    となり、

    std::map<std::string, std::string> ret2 = getX3DShapeNames("apple");
    ret2[":"] -----> "apple.wrl"

    となる．
